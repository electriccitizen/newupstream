Web Accessibility

Web Accessibility is a huge push in modern web development and design. The web should be equally accessible to all people—no matter they ability or disability. Citizen Patterns strives to make every site as accessible as possible.

Since Electric Citizen works primarily in the public and university sector—accessibility double important to both us and our clients. We are constantly striving for improvement in any area of accessibility.

Key Development Areas for Accessibility:
  • Color Contrast
  • Aria elements
  • Image Alt Tags
  • Keyboard Accessibility for all menus, links, buttons, expanding elements
  • Jump Link for main content and menus

Accessibility Color Contrast

According to web standards, text must have a minimum color contrast of 4.5:1 between the background color and the text. Color contrast can be tested by using any of various online tools.

Design Mockups normally address color contrast, but it is the responsibility of every developer to double check colors for contrast levels and to account for and theme proper contrast levels for elements not covered in mockups.

Hover states and functional elements should strive to have an accessible level of contrast, but they are not quite as important as the default settings.

If a mockup color needs to change to account for contrast or something not mocked up changes the overall design, the designer should be consulted before making the change.

Aria Standards

Aria standards should be used whenever possible. Drupal assigns many of these standards in the core templates and functionality but it is the responsibility of the developer to implement them when customizing the site.

Aria Standards:

  • Aria Labels for all form elements
  • Unique identifies for all elements
  • Roles to describe the structure of layouts.
  • Properties to describe the state of elements (checked, aria-expanded, has-popup, etc)
  • A way to access these elements through the keyboard.

Other Accessibility Standards and Best Practices

  1. Alt Tags— All images on the site are required to have an alt tag. This is automatically done through the Image media bundle. For background images on sections or widgets set as inline styles through twig, add the alt text as an aria-label on the element using the background image and add an img role.
  2. Keyboard Accessibility— All clickable elements and regions should be keyboard accessible if at all possible. This can be done through button elements or roles, jump links to sections/regions, and tab indexes (only when absolutely needed.
  3. Aria Elements— All aria elements should be used whenever possible, include roles, aria-labels, and aria-expanded attributes.
  4. Color Contrast— All text colors should be tested for contrast both by the designer and the developer.
