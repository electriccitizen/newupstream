Troubleshooting

Citizen Patterns normally is pretty bulletproof if you are using the most recent versions of npm and node.js. If you do run into trouble when either installing the theme or running (npm start), follow these steps:

  1. Update to the most recent version of npm and node.js— Since this theme is based on Four Kitchens Emulsify it is best to run the most current versions of both npm and node.js in order to stay even with dependency needs.
  2. Make sure non of you files are misnamed or contain incorrect syntax— ES Lint (JS Linting) will cause the theme to quit running if it detects a JS error. It will give you the error and line to fix. New files (SCSS, JS & Twig) will also often cause the theme to quite running. Clear the site cache and re-run npm start. Twig syntax/variable errors will often cause a white screen with a fatal error message. use the Recent Log Messages report (Config > Reports) to get the specific Twig error, fix it and clear cache.
  3. Check the error to see if it has to do with theme dependencies— Sometimes theme dependencies will fail due to npm/Gulp developers falling behind or version conflicts. Often resolutions to dependency errors can be found in issue queues either in the Emulsify theme repo, the Emulsify Gulp repo, or on the npm package or dependency repo issue queue.
  4. Scream for help at the top of your lungs— When all else fails, the Lead Front-End developer and/or Technical Director can most likely find a solution to your problem.
