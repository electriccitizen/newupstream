WYSIWYG Styles

CKEditor (WYSIWYG) styles should be updated to match the text, headers and any other inline styles required by each site. WYSIWYG editor styles should match the front-facing output as closely as possible.

Updating existing styles— Existing styles can be changed/removed in the cke.css stylesheet in /components/css. When changing styles, you may need to clear site cache several times or even clear an hour or so of your browser history for the changes to take effect. WYSIWYG font calls can be updated in the citizen_patterns.info.yml. See Site-specific Colors & Fonts for more information.

New WSYIWYG styles— New WYSIWYG styles can be added through custom classes set in Configuration > Text Formats and Editors. To add a new style, configure the editor and add your style in the Styles Dropdown input area follow the existing formats. If you are adding a style to a new type of element (ul, drupal entity, img, etc) make sure that element is allowed to have styles in the Basic HTML Limit allowed HTML tags section at the bottom.

Once your new style is added to the editor, add the style to the cke.css stylesheet and the appropriate theme Scss file (usually 01-Atoms/02-text/_text.scss). Clear cache (and possibly your recent history) to get the style to show up.
