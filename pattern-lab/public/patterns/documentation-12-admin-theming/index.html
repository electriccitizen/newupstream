Citizen Module

The Citizen module is a module that is set for theming the admin side of Drupal 8 and performing other non-theme related tasks. The module is stored in web/modules/custom/citizen.

Admin CSS— Admin CSS is handled through two files in Citizen: global.css and node.css. CSS files are used because installing node and Gulp and compiling Scss files for the small amount of code used in these files is a bit of an overkill.

Global.css is used for all CSS outside of the node/add and node/edit forms (admin views, select lists, media browser and the admin toolbar. Node.css is used for all node related admin theming (field spacing, paragraph buttons, help text, labels, etc).

Admin JS— Admin JS is also stored in two files: global.js and node.js. Global.js is attached to every admin page using the citizen_custom.libraries.yml and an admin page-alter function in the .module file. The global library is also used to call the Select2 JS file on all select lists on the site (both admin and front-facing). Select2 makes select lists easier to style and adds autocomplete and drop-up/drop-down functionality.

Node.js is used for showing and hiding node form fields and other tasks that will effect the editing experience. Some fields, for instance, should only be shown if another field is used or has a certain value selected. Node.js fills that need. NOTE: Any field that is dependent on the output of another in node.js should have that same output dependency required on the front-end (through Twig, JS, or Scss).

Paragraph Admin Icons

Each paragraph button must be given an icon and themed in the Citizen module custom-node.css file. The icon is stored in the Citizen Custom module /icons/svg folder. Icon ai files are stored in the Electric Citizen Google Drive zFront End Elements folder. Use the widget-admin-icons.ai file to design the icon and the widget-icons-svg.ai file to size it and export it to the custom module icons folder. The buttons are themed and named based on the paragraph bundle name following the conventions already in use in Citizen node.css.

Paragraph bundles also need to have the admin icon uploaded to both your local site and the current non-local development platform. This can be done by editing the paragraph bundle and uploading the icon in the field provided. Your local settings should then be immediately exported to the development environment. This must be done because these file icons are considered both content and configuration so the files must be added in both areas to get the correct file number in config—which is then exported to sync the config so the icon is properly stored in both places.

WYSIWYG Styles

CKEditor (WYSIWYG) styles should be updated to match the text, headers and any other inline styles required by each site. WYSIWYG editor styles should match the front-facing output as closely as possible.

Updating existing styles— Existing styles can be changed/removed in the cke.css stylesheet in /components/css. When changing styles, you may need to clear site cache several times or even clear an hour or so of your browser history for the changes to take effect. WYSIWYG font calls can be updated in the citizen_patterns.info.yml. See Site-specific Colors & Fonts for more information.

New WSYIWYG styles— New WYSIWYG styles can be added through custom classes set in Configuration > Text Formats and Editors. To add a new style, configure the editor and add your style in the Styles Dropdown input area follow the existing formats. If you are adding a style to a new type of element (ul, drupal entity, img, etc) make sure that element is allowed to have styles in the Basic HTML Limit allowed HTML tags section at the bottom.

Once your new style is added to the editor, add the style to the cke.css stylesheet and the appropriate theme Scss file (usually 01-Atoms/02-text/_text.scss). Clear cache (and possibly your recent history) to get the style to show up.
