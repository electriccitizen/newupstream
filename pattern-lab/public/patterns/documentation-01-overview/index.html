Front End Tools/Requirements

  • Sublime Text (or similar text editor)
  • iTerm (recommended)
  • Tower Git Client (recommended)
  • Adobe Creative Suite (Photoshop / Illustrator)
  • NPM (min version 5.5.1)
  • Node.js (min version 6.11.2)
  • Virtual Machine requirements specified in the site repo ReadMe onboarding section.
  • GitHub account and access to the site repo.
  • One Password password manager (recommended)

Rules of the Road

Push early and often— All EC D8 sites use a GitHub based workflow. Pushing early in a project and pushing frequently will help prevent you from falling behind other developer's working on the same repo and will make a rollback to an earlier commit easier if necessary.

Theme mobile first— All page mockups come with a mobile version (and often a tablet sized version as well). It is almost always best and easiest to theme from the mobile version to the desktop version as most of the theme breakpoints run from small to large. Mobile versions are also usually less intricate than desktop so you can theme additively rather than subtractively.

Bite-sized theming— EC follows a component (atomic) approach to theming. We always try to theme the smallest chunks first and them put them together to complete the overall design. This helps ensure consistency while limiting repetition.

Everyone loves Chatty Cathy— All of your Sass, Twig and JS awesomeness might make perfect sense in your head while building, but for another developer (or even future you from 6 months from now) it can often be very painful and time consuming to pick apart what you were thinking. Comments are the best way to clear the path to easy and efficient development. The styles of commenting used by EC in each type file are covered in the best practice section for each type of theming.

Use all the things— Designs usually come with both .ai files and .jps. Its often most helpful to get all the sizes, colors and overall feel from the jpgs using Photoshop and then to do the type sizes using the Illustrator files.

Pattern Lab Overview

Pattern Lab is a tool for creating atomic-based style guides for websites. Citizen Patterns uses Pattern Lab as its Drupal theme engine for template inheritance, Sass & JS compiling and other theme tasks, but Citizen Patterns is not meant to be a style guide that is passed on the to the client. At this time it is a tool for themer's only and as such is not deployed to git or viewed outside of a local development atmosphere.

Citizen Patterns is based on the Emulsify theme developed by Four Kitchens and as such we do not have 100% control over all node module aspects of the theme, but very little of the original Emulsify theme implementation is still left.
